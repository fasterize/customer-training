Comment identifier et corriger un problème ?

Une page (ou le site complet) est cassée

Utiliser les paramètres de debug en ajoutant les paramètres suivants dans l'URL :

  • ?frz-all=false pour tout désactiver sur une page
  • ?frz-[rule]=false pour désactiver une règle sur une page
  • ?frz-[rule]-[option]=false pour désactiver une option d'une règle sur une page
  • ?frz-persist-[rule]=false pour désactiver une règle sur une page et sur la session
  • Voir doc en ligne pour la liste des règles

Une page (ou le site complet) est cassée

Une fois la règle posant problème identifiée, modifier la configuration pour corriger le problème :

  • Activer / Désactiver des règles

    Les règles qui posent problème sont en général

    1. DeferJS
    2. Concaténation
    3. Minification
    4. Lazyloading

    Les règles peuvent désactivées pour des pages données ou globalement

  • Exclure une ou plusieurs URL qui pose problème au niveau global
  • Ajouter un nom de domaine secondaire

Une page d'erreur s'affiche : 'Couldn't find origin server...'

  • L'alias n'est pas configuré dans Fasterize
  • Il faut le rajouter dans la configuration des domaines

Les images sont trop dégradées

Certaines images, notamment certains dégradés, peuvent subir une modification trop visible

  • Exclure les images concernées en les mettant en blacklist de la compression d'image
  • Exclure les images concernées de façon globale
  • Désactiver PNG24 PNG8
  • Désactiver la compression des images
